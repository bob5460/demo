流程开始

ERP添加待复核单据到单据复核表

ERP发送消息给复核人

单据复核流程

返回流程列表

复核人进入复核待办

结束

1、单据复核是指在单据事后进行的确认操作。如前台销售单打折复核。


如果需要进行打折复核，则ERP在单据过账成功后，将复核的单据信息插入到tbsys_djfh表。

通过/驳回

系统发送消息通知提交人复核结果

无论通过或是驳回，均由erp存储过程进行后续的处理。应用对结果不作干预或特殊的处理。

提交人收到后，点击进入查看详情。

/**单据复核**/

CREATE TABLE tbsys_djfh(

     f_id INT NOT NULL IDENTITY(1,1),                    --自增id

     f_djid INT NOT NULL,                                   --单据id

     f_mkid INT NOT NULL,                                   --模块id

     f_djhm VARCHAR(30) NOT NULL,                         --单据号码

     f_bmid INT NOT NULL,                                   --部门id

     f_bmmc VARCHAR(100) NOT NULL,                         --部门名称

     f_fhrid INT NOT NULL,                                   --复核人id

     f_fhrmc VARCHAR(30) NOT NULL ,                         --复核人姓名

     f_fhzt VARCHAR(10) NOT NULL DEFAULT 'pending',     --待审核pending，审核通过passed,审核驳回rejected

     f_fhyj nvarchar(255) NOT NULL DEFAULT '',          --复核意见

     f_fhsj DATETIME null,                                   --复核时间

     f_sqrid INT null,

)

go;
